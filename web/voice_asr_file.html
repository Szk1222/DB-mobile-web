Overview · 产品介绍

关于录音文件识别

录音文件识别（Audio File Recognition）是面向已录制音频的非实时批量转写服务。用户上传音频文件后，引擎异步完成解码与识别，并回调返回带时间戳的完整转写结果。

产品支持 WAV、MP3、M4A、AAC、FLAC 等主流音频格式与多种采样率，单文件最长可达 5 小时，并支持批量任务提交。识别结果包含分句时间戳、说话人标注与置信度信息，便于后续检索与剪辑对齐。

相比实时识别，录音文件识别更注重转写完整度与准确率，适合通话录音归档、音视频内容字幕生成、访谈笔录整理等对时延不敏感、对质量要求高的离线场景。

Core Features · 核心功能

核心能力一览

多格式批量识别

支持 WAV / MP3 / M4A / AAC / FLAC 等格式，单文件最长 5 小时，可批量提交。

分句时间戳

为每个识别分句输出起止时间戳，便于字幕对齐与音频精准定位。

多说话人标注

自动区分录音中的不同说话人，按角色组织通话与会议文本。

结构化输出

返回包含文本、时间、说话人、置信度的结构化结果，可直接入库检索。

异步回调机制

任务异步处理，支持回调通知与轮询查询，长音频转写无需保持连接。

热词与顺滑

支持领域热词表与口语顺滑，提升专有词识别率并输出规范文本。

API Reference · 接口说明

快速对接接口

核心请求参数与调用示例，便于快速完成接入

参数类型必填说明
app_idstring必填应用 ID，开放平台创建应用后获取
api_keystring必填接口鉴权密钥
audio_urlstring必填音频文件的可访问 URL 地址
audio_formatstring必填音频格式：wav / mp3 / m4a / aac / flac
languagestring选填识别语种，默认 zh-CN
enable_speakerbool选填是否开启多说话人分离，默认 false
enable_timestampbool选填是否输出分句时间戳，默认 true
callback_urlstring选填识别完成后的结果回调地址
 
# 1. 提交识别任务
curl -X POST "https://openspeech.data-baker.com/asr/v1/file" \
  -H "Authorization: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
        "app_id": "your_app_id",
        "audio_url": "https://your-bucket/meeting.mp3",
        "audio_format": "mp3",
        "enable_speaker": true,
        "enable_timestamp": true
      }'
# 返回 task_id
