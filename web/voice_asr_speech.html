Overview · 产品介绍

关于长语音识别

长语音识别（Long Speech Recognition）面向不限时长的连续语音输入，提供持续稳定的实时流式转写能力。引擎以 WebSocket 长连接接收音频流，边接收边解码，并随说话进度持续返回转写结果。

产品在一句话识别的声学能力基础上，强化了长时段语音的稳定性与一致性：自动断句分行、标点预测、口语顺滑（去除语气词与重复），并支持多说话人分离，为每段文本标注说话人角色。

无论是数小时的会议录音实时转写、课堂直播字幕，还是呼叫中心的全程通话记录，长语音识别都能以低延迟稳定输出规范文本，并提供公有云 API 与私有化部署两种接入方式。

Core Features · 核心功能

核心能力一览

持续流式转写

WebSocket 长连接持续接收音频流，不限时长边说边出字，长时段转写稳定无中断。

自动断句与标点

按语义自动断句分行并预测标点，长文本输出结构清晰、便于阅读与归档。

多说话人分离

区分对话中的不同说话人并标注角色，会议与通话记录可按发言人组织。

口语顺滑处理

自动去除“嗯”“啊”等语气词与口吃重复，输出更接近书面表达的规范文本。

多语种与方言

支持普通话、英语、粤语等 20+ 语种方言，并可识别中英文混读长段落。

热词与领域适配

支持上传领域热词表，提升专业术语、人名地名等专有词汇的长文识别准确率。

API Reference · 接口说明

快速对接接口

核心请求参数与调用示例，便于快速完成接入

参数类型必填说明
app_idstring必填应用 ID，开放平台创建应用后获取
api_keystring必填接口鉴权密钥，请妥善保管
audio_formatstring必填音频编码：pcm / wav / mp3 / opus / speex
sample_rateint必填采样率，取值 8000 或 16000
languagestring选填识别语种，默认 zh-CN
enable_punctuationbool选填是否添加标点，默认 true
enable_speakerbool选填是否开启多说话人分离，默认 false

import websocket, json, threading

URL = "wss://openspeech.data-baker.com/asr/v1/long"
ws = websocket.create_connection(
    URL, header=["Authorization: your_api_key"])

ws.send(json.dumps({
    "app_id": "your_app_id",
    "audio_format": "pcm",
    "sample_rate": 16000,
    "language": "zh-CN",
    "enable_punctuation": True,
    "enable_speaker": True
}))

def receiver():
    while True:
        msg = json.loads(ws.recv())
        if msg.get("is_final"):
            break
        print(msg["result"]["text"])

threading.Thread(target=receiver).start()

# 持续推送音频分片（每 200ms 一帧）
with open("meeting.pcm", "rb") as f:
    for chunk in iter(lambda: f.read(6400), b""):
        ws.send_binary(chunk)
ws.send_binary(b"")
